# API 版本控制策略

## 当前版本

当前 API 版本为 **v1**，所有接口前缀为 `/api/v1`。

## 版本控制规则

### 1. 向后兼容变更（不需要新版本）
- 新增 API 端点
- 新增可选请求参数
- 新增响应字段

### 2. 破坏性变更（需要新版本）
- 删除或重命名 API 端点
- 删除或重命名请求/响应字段
- 修改字段类型
- 改变认证机制

### 3. 版本迁移策略
- 新版本发布后，旧版本至少维护 **6 个月**
- 旧版本返回 `Deprecation` 和 `Sunset` 响应头
- 在新版本上线前 3 个月通知所有 API 使用方

## v2 迁移规划

### v2 预期变更
- 支持多租户架构
- 自定义分析模型接口
- WebSocket 实时推送
- 批量操作 API

### 迁移步骤
1. 发布 `/api/v2` 路由，与 `/api/v1` 并行运行
2. `/api/v1` 响应添加 `Deprecation: true` 头
3. 在 v1 文档中标注每个端点的 v2 替代方案
4. 6 个月后评估 v1 使用量，决定是否下线

## 路由结构

```
/api/v1/auth/*       — 认证相关
/api/v1/chat/*       — 对话查询
/api/v1/reports/*    — 报告管理
/api/v1/analysis/*   — 数据分析
/api/v1/agents/*     — 智能体
/api/v1/upload/*     — 文件上传
/api/v1/voice/*      — 语音接口
/api/v1/data/*       — 数据集成
```
